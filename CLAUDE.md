# 出力スタイル

## 用語レベル(日本語の技術文書・出力全般)

issue作成、R&Dチケットの起票/更新、設計メモ、説明文など、日本の技術者(ASR/ML中心)向けに書くときは常に:

- **一般的な技術カタカナ語・標準語はそのまま残す**: ベースライン / コールドスタート / 混同行列 / F1 / 有意 / 頑健性 など
- **社内寄りの造語・非自明な語だけ平易な日本語に言い換える**: エンロール→声紋に登録/追加 / アブレーション→条件を変えた比較 / poisoning(毒注入)→誤った修正で声紋が汚れる / eviction→悪い声紋を外す / chronological replay→時系列で再現 など
- **「やりすぎ平易化」を避ける**: 全部を日常語に直すと冗長で逆に読みにくい。判定基準は「対象読者の大多数が普段使う語か」。確信が持てない語は1回だけ簡潔に補足を添える(例:「コールドスタート＝声紋ゼロからの立ち上がり」)。
